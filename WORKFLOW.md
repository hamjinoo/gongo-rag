# WORKFLOW

이 프로젝트의 학습 목표는 모든 코드를 손으로 입력하는 것이 아니라, **각 부품의 입력·출력·실패 원인·선택 이유를 설명하고 실험 결과로 판단하는 것**입니다.

## 한 번의 작업 단위

1. 해결할 실패나 검증할 가설을 한 문장으로 적는다.
2. 기준선 결과를 저장한다.
3. 한 번에 하나의 변수만 바꾼다.
4. 테스트와 같은 평가셋을 다시 실행한다.
5. 좋아진 점, 나빠진 점, 대표 실패를 기록한다.
6. 유지할지 되돌릴지 결정한다.

## 직접 판단할 것

- 골든셋의 질문·정답·출처가 맞는가?
- 어떤 지표로 변경을 채택할 것인가?
- 검색 실패인가, 답변 실패인가, 원문 문제인가?
- 이 기술이 지금 필요한가?
- 결과와 한계를 내 말로 설명할 수 있는가?

## AI에게 맡겨도 되는 것

- LangChain·Chroma·reranker·LangGraph 연결 코드
- 반복적인 데이터 변환과 설정 파일
- 테스트·평가 실행 코드와 결과표 생성
- FastAPI·Docker·CI 같은 서비스 배관
- 코드 리뷰, 디버깅, 문서 구조와 문장 다듬기

AI가 작성한 코드도 다음 네 질문에 답할 수 있어야 완료입니다.

1. 입력과 출력은 무엇인가?
2. 실패하면 어디서 확인하는가?
3. 이 부품을 끄면 기준선으로 돌아갈 수 있는가?
4. 테스트와 평가 명령은 무엇인가?

## 완료 기준

기능이 실행되는 것만으로 끝내지 않습니다. 같은 평가셋에서 숫자를 비교하고, 대표 실패 사례와 기술 선택 이유를 README 또는 `experiments/`에 남기면 완료입니다.
